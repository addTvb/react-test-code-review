# Задание студента, которое вы будете проверять

Необходимо реализовать сервис хранения списка задач по проектам. Для бэкенда списка задач используется API сервиса Todoist. Запросы авторизации отправляются на бэкенд auth.nomoreparties.co. У всех пользователей будут общие задачи и проекты.

Механизм работы и требования:

- При отправке формы регистрации должен отправляться запрос на адрес `/signup`.
- После успешной регистрации должен происходить редирект на страницу входа.
- При отправке формы входа должен отправляться запрос на `/signin`.
- После успешного входа должен выполняться редирект на страницу `/projects`. Только авторизированные пользователи могут попасть на страницу `/projects`.
- На странице проектов пользователь может переключаться между проектами и просматривать задачи каждого из них.
- Если пользователь не авторизован, то при попытке перейти на страницу `/projects` его перенаправляет на `/signin`.
- В шапке сайта должен отображаться email авторизированного пользователя.
- При нажатии на кнопку «Добавить» должен отправляться запрос на создание новой задачи. После ответа сервера поле ввода должно очищаться.
- При нажатии на кнопку «Удалить» должен отправляться запрос на сервер и задача должна удаляться со страницы.
- При нажатии на кнопку «Выход» пользователь выходит и перенаправляется на страницу входа.

- Корректно ли построена логика запросов к серверу?
- Корректно ли реализован роутинг между страницами?
- Всё ли нормально с JSX и разметкой?
